### 정의

여러 개의 프로세스가 진행 중일 때, 기존에 실행 중인 프로세스를 중단하고 다른 프로세스를 실행하는 것을 의미합니다. 즉, CPU 에 실행할 프로세스를 교체하는 것입니다.

### PCB

컨텍스트 스위칭은 PCB 라는 메모리의 별도 공간에 프로세스 정보들을 저장해놓고 그것을 찾는 방식으로 진행됩니다. PCB 에 저장되는 내용은 아래와 같습니다.

- 프로세스 ID (PID)
- 레지스터 값 (SP, PC)
- 스케줄링 정보 (프로세스 상태)
- 메모리 정보
- 기타 정보들

### 컨텍스트 스위칭 동작 순서

A 프로세스가 실행 중이고 B 프로세스가 대기 상태라고 합시다.

- 스케줄러가 A 를 중단하고 B 를 실행할 것을 요청
- A 프로세스의 스택의 데이터 위치를 가리키는 SP 와 다음 실행할 코드의 주소를 가지는 PC 값을 PCB 에 저장함
- A 프로세스가 대기 상태로 변경되고, CPU 가 B 를 실행함.
- 반대로 B 에서 A 로 컨텍스트 스위칭할 경우 B 정보들을 PCB 에 저장하고, 아까 저장한 A 프로세스의 PCB 에서 값을 찾아와 덮어씌움.
