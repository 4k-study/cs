## ❓ 질문
### ******Q. 아사 현상과 데드락의 차이점이 뭔가요?******
```
아사 현상은 스케줄링 정책이 공평하지 못해서 오랜 기간동안 프로세스가 자원을 할당받지 못하는 상황을 말하고 이는 정책을 보완하여 충분하게 해결할 수 있습니다.
데드락은 두 개 이상의 프로세스가 서로 점유하고 있는 자원을 무한히 대기하는 상황을 말하고 이는 정책이나 알고리즘을 해결하기가 어렵습니다.
```
<br>

### ******Q. 데드락 예방의 단점이 뭔가요?******
```
예방 방식은 시스템 처리량이나 자원 사용 효율이 떨어지는 단점이 있습니다.
```
<br>

### **Q. 데드락이 생기는 경우 예시 좀 들어주세요**
```
DB 에서의 상호 거래 패턴을 예시로 들 수 있습니다.
A 와 B 유저가 서로 돈을 교환하려고 합니다. A 는 B 에게 3000원을 주고 B 는 A 에게 1000원을 주려고 하는 상황입니다.
이 때, 이 둘의 교환이 동시에 이루어지는 경우 데드락이 발생합니다.
A 와 B 가 서로 Lock 처리된 상대방의 자원을 요청하기에 데드락이 발생하는 것입니다.
```
<br>

### **Q. 회피기법 중 은행원 알고리즘 설명해주세요**
```
프로세스가 자원을 요구할때 자원을 할당한 후에도 안정 상태이면 자원을 할당하고, 그렇지 않으면 다른 자원이 해제될때까지 대기했다가 자원을 할당하는 방식입니다.
```
<br>

### **Q. 데드락을 어떻게 회복 시키는지 구체적인 방법을 하나 들어주세요**
```
데드락에 있는 모든 프로세스를 중지하거나, 데드락이 해결될 때까지 프로세스를 하나씩 중지하는 방법이 있습니다.
혹은, 프로세스들로부터 자원을 빼앗아 데드락이 해결될 때까지 다른 프로세스에 자원을 할당하는 방법도 있습니다.
```
